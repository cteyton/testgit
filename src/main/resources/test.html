Themis propose deux types de profil : manager et développeur. 

Un profil manager est un généralement profil chef de projet, lead developer ou encore Scrum Master. Ce profil a accès à toutes les actions de développement, en fonction des permissions accordées sur les projets et les équipes. A l'inverse du développeur, ce profil n'effectue pas d'actions de développement. 

Un profil développeur est attribué à toute personne susceptible d'effectuer des actions de développement (modification de fichier, ...). Un profil doit être associé à un seul auteur afin que toutes les actions effectuées par celui-ci seront soient sur l'utilisateur-trice. Si plusieurs auteurs correspondent à un-e utilisateur-trice, vous devrez au préalable fusionner ces auteurs pour n'obtenir qu'un seul auteur. 

Compte manager

Chaque action dans Themis est associée à un projet et une équipe. Ainsi, il est nécéssaire d'associer à un profil manager un ensemble de projets et d'équipes pour qu'il puisse accéder à ces actions. Si aucun projet n'est associé à un profil manager, ce dernier obtiendra alors un Themis vide de donneés lors de sa connexion. 

Les permissions associées aux managers sur les projets et les équipes peuvent être respectivement gérées dans la vue Projets et Equipes au moment de la création ou de l'édition. Mais il est possible également de gérer individuellement les permissions d'un-e manager. Pour cela, rendez-vous dans la vue Administration puis dans la vue Utilisateurs. Identifiez le manager que vous souhaitez mettre à jour, puis cliquez dans le colonne Actions sur le bouton Editer :

Dans le formulaire qui s'ouvre, identifiez la section Gestion des permissions dans laquelle sont listées les équipes dans la partie gauche, et les projets dans la partie droite. Les éléments autorisés sont marqués d'une case cochée et la ligne est surlignée. Vous pouvez ainsi cocher et décocher les cases en fonction des permissions que vous souhaitez mettre à jour. 

Pour terminer l'opération, cliquez sur le bouton Valider.

Compte développeur

Mettre à jour l'auteur associé aux utilisateurs

Pour mettre à jour l'auteur associé, rendez-vous dans la partie Administration, puis l'onglet Utilisateurs. Dans la liste des utilsateurs, identifiez la ligne correspondant à l'utilisateur que vous souhaitez mettre à jour, puis cliquez sur le bouton Editer 

Ensuite, vérifiez dans le formulaire que le profil de l'utilisateur-trice est marqué comme Développeur.

Dans le champ suivant, il vous sera demandé d'associter l'utilisateur-trice à un auteur.

Recherchez alors le nom de l'auteur pour que le mécanisme d'auto-complétion vous suggère des résultats. Sélectionnez ensuite le résultat que vous souhaitez, puis valider l'opération en cliquant sur le bouton Valider situé en bas du formulaire.

Ajout de tags pour la fusion automatique

Un auteur est une identité unique sur une source-todo.  Au fur et à mesure que les projets avancent et que de nouveaux débutent, il sera nécéssaire de créer de nouvelles sources au sein de Themis. Si des utilisteurs-trices de Themis ayant un profil "développeur" travaillent sur ces sources, de nouveaux auteurs seront crées. Dans ce cas, il faudra les fusionner avec les auteurs existants pour mettre à jour l'association avec ces utilisateurs-trices. 

Néanmoins, il est possibile d'anticiper cette opération en indiquant pour chaque utilisateur-trice une liste de tags. Un tag est une chaîne de caractère qui sera recherchée parmi les identités trouvées sur une source, et toute correspondance exacte génèrera une association automatique entre l'utilisateur et ce nouvel auteur. 

Pour mettre à jour l'auteur associé, rendez-vous dans la partie Administration, puis l'onglet Utilisateurs. Dans la liste des utilsateurs, identifiez la ligne correspondant à l'utilisateur que vous souhaitez mettre à jour, puis cliquez sur le bouton Editer  :

Dans la partie du formulaire associée aux tags, indiquez tous les identifiants associés à l'utilisateur-trice.Terminer l'opération en cliquant sur le bouton Valider situé en bas du formulaire.

Prenons un exemple pour clarifier. L'utilisateur Jon Snow est actuellement associé à l'auteur jsnow, lui-même fusionné avec jonsnow. Une nouvelle source est ajoutée dans Themis, et Jon a pour mission d'effectuer des tâches dessus. Pour anticiper la fusion des auteurs, il met à jour son profil en indiquant les tags qu'il pense être les identifiants qu'il utilise régulièrement. 

Lors du prochain rafraîchissement des données de Themis, un auteur est trouvé sur la nouvelle source avec pour nom jon.snow@nightwatch.north. Que se passe-t-il ensuite ? L'auteur jsnow est fusionné avec cet auteur, de telle sorte que notre utilisateur est virtuellement associé à 3 auteurs différents. Dans le cas où il n'était pas encore associé à un auteur, Jon Snow aurait été associé avec le nouvel auteur jon.snow@nightwatch.north. 

Ainsi, il est recommandé à chaque profil développeur de remplir ces champs pour anticiper la fusion des auteurs, surtout si votre organisation compte un nombre important de sources.
